# Star Citizen Blueprint Extractor

A lightweight, real-time Python utility designed for **Star Citizen** to automatically scan, track, and log newly acquired crafting blueprints from your local game files. 

It keeps a persistent record of your blueprint collection in a `.csv` file, tracks exactly *when* you found them, and monitors live game logs so you never miss an acquisition.

---

## Features

* **Live Monitoring (`tail -f` style):** Actively watches your `Game.log` file while you play. As soon as a blueprint is registered by the game client, it catches it and updates your log.
* **Log Rotation Detection:** Automatically detects if the game client restarts or clears `Game.log`, seamlessly reopening the file without needing a script restart.
* **Historical Backup Scanner:** Option to scan older archived logs (`logbackups`) to retroactively compile your existing blueprint collection.
* **Human-Readable Timestamps:** Displays your 10 most recently acquired blueprints on startup with a relative time breakdown (e.g., *“3 min ago”*, *“1 hr ago”*, *“2 days ago”*).
* **Duplicate Prevention:** Cross-references with your local `blueprints.csv` to ensure no entry is duplicated.

---

## Prerequisites

* **OS:** Windows (Requires the `msvcrt` library, which is built into Python for Windows platforms).
* **Python:** Python 3.6 or higher.

---

## Installation & Setup

1. **Download the Script:** Save `blueprint_extractor.py` to a directory of your choice.
2. **Verify Star Citizen Path:** By default, the script points to the standard RSI installation path:

```python
   LIVE_PATH = "C:\\\\Program Files\\\\Roberts Space Industries\\\\StarCitizen\\\\LIVE"
```

*If your game is installed on a different drive or directory, open the script in a text editor and update the `LIVE_PATH` variable accordingly.*

---

## How to Use

1. Open your terminal or Command Prompt (`cmd`).
2. Navigate to the folder containing the script.
3. Run the script using Python:
```bash
python blueprint_extractor.py

```



### Step-by-Step Walkthrough

* **On Launch:** The script will prompt you regarding older log backups:
```text
Would you like to scan older log backups for blueprints? (y/n):

```


* Choosing `y` or `yes` will read your `logbackups` folder (filtering for logs modified after March 24, 2026) to scrape any historical blueprints you might have missed.


* **The Dashboard:** It will then load your current database and display the 10 most recent acquisitions:
```text
================================================================================
ACQUIRED             | BLUEPRINT NAME
--------------------------------------------------------------------------------
3 min ago            | Pyro RYT Multi-Tool
1 hr ago             | Gemini C54 SMG
2 days ago           | Behring P4-AR
================================================================================

```


* **Live Tracking:** The script will sit in a low-resource idle loop. Play Star Citizen as usual. Whenever you acquire a blueprint, it will print directly to the console and save instantly. This script must be restarted each time to start Star Citizen.
* **To Exit:** Press the **`Escape`** key or **`Ctrl + C`** in the command prompt window to safely stop the logger.

---

## Data Storage

The script automatically generates a `blueprints.csv` file in the same directory where the script is located. The file is structured as follows:

```csv
Blueprint Name,Timestamp
Pyro RYT Multi-Tool,2026-05-25T10:15:30Z
Gemini C54 SMG,2026-05-25T09:45:12Z
```

## Troubleshooting

* **"Live log not found" Error:** This means the script cannot find your `Game.log`. Double-check your `LIVE_PATH` variable inside the script to ensure it accurately reflects where your game is installed.
* **Script isn't catching new blueprints:** Make sure you are actively playing on the account/machine where the script is running. The script relies entirely on game log output generated by your local Star Citizen client.
* **I'm seeing duplicates with a different name:** This will happen if you use different localisation files (e.g. [StarStrings](https://github.com/MrKraken/StarStrings)). This script will pull BPs from the log as you see them in your version of the game. Not from the game files or your database of BPs on the server.
